What to do next

Most departments create one request per event cycle, then revise until approved.
1) Start a request
Pick an event cycle and department. Add line items in the draft.
Create new request
DRAFTYou can edit and add lines.
SUBMITTEDUnder review. Editing is locked for requesters.
NEEDS_REVISIONKicked back. Update the draft and resubmit.
APPROVEDFinalized and locked.
Org-level requests (future)
Requests not tied to an event (MAGit, org-wide tools, shared infrastructure) will be supported in a later phase. This MVP focuses on event + department requests.

Important notes
  • If your department’s budget is already approved for an event cycle, the system blocks creating a second base request and will prompt for a change request workflow (future).
  • Use clear “Reason” text on each line. Reviewers rely on it.
